ToolFinder

This page lists bioinformatics tools installed across several of the BioCommons infrastructure partner systems and tool installation request mechanisms on these systems. Tool status on other partner infrastructures will be added over time.

Note:

  • Table last updated 2021-09-30.
  • Software documentation is linked from the Tool / workflow name in the first column.
  • The Tool identifier column contains an identifier for the tool / workflow: typically either the module name (used for matching to HPC lists) or a placeholder value.
  • The EDAM topics column categorises the tools by purpose, using an EDAM term where possible.
  • For tools which are not currently installed on Galaxy Australia, but which are available in the Galaxy app store (aka toolshed), the Available in Galaxy toolshed column launches a toolshed search using the link label as the search term.
  • More information about a tool can be found by following the link in the bio.tools documentation column.
  • When a tool has been containerised to allow for easier installation on any compute infrastructure, a link to the containerised software that can be downloaded from BioContainers is shown in the Containers available? (BioContainers) column.
  • The list currently includes some tools that are not used for bioinformatics.
  • The source material for the table is currently manually curated, and while we endeavour to keep the information as current as possible, there is a natural limit to the volume of information maintained here. Production of this information will be automated during 2021, and tools that are not relevant for bioinformatics analyses removed.

FAQ: How can I get a HPC allocation?

Allocation of computational resources to individual researchers on partner HPC and cloud infrastructures is through the usual channels offered via each partner (see instructions for each by following these links: NCI, Pawsey, QCIF, ARDC’s Nectar Research Cloud). The BioCommons BioCloud activity is working with these partners to examine ways to make it easier for life science researchers to receive HPC and HTC allocations.

Note that access to computational resources via the Galaxy Australia hosted web platform is unlimited for Australian researchers thanks to our computational partners (ARDC’s Nectar Research Cloud, University of Queensland, QCIF, University of Melbourne and Pawsey) who underpin the Galaxy Australia service.

How were the tools and workflows selected?

The tool list here includes those that have been identified through BioCommons Community consultations and other engagements, as well as through matching to the tool sets installed across the various BioCommons partner compute infrastructures. The list is not intended to be exhaustive: i.e. it does not contain ALL bioinformatics tools.

How was the list generated?

RStudio (see session info at the bottom of this page) was used for processing.

The broad steps are described here:

  1. The manually curated tool list and separate tool lists provided by the BioCommons partner infrastructures were parsed (Galaxy Australia, NCI, Pawsey and QRISCloud - RCC).
  2. bio.tools metadata was imported for those tools where a biotoolsID was available.
  3. The complete set of tool lists from the BioCommons partner infrastructures were joined using the toolID as a key, and
  4. Links were embedded, where available, to the original documentation URLs (homepage or GitHub repository, for example) as well as the registry entries on bio.tools and BioContainers.
  5. Galaxy wrapping often entails that tool suites are separated into their individual tool components. As such, if a Galaxy tool was matched to the manually curated tool list, a tool shed search link was embedded in the Available in Galaxy toolshed column. This allows a user to search for the tool using a relevant search term (e.g. vcftools) to identify all the component tools available in the toolshed.
