FAQ: can I get a HPC allocation through the BioCommons?

Allocation of computational resources to individual researchers on partner HPC and cloud infrastructures is through the usual channels offered via each partner (see instructions for each by following these links: NCI, Pawsey, QCIF, ARDC-Nectar). The BioCommons BioCloud activity is working with these partners to examine ways to make it easier for life science researchers to receive HPC and HTC allocations.

Note that access to computational resources via the Galaxy Australia hosted web platform is unlimited for Australian researchers thanks to our computational partners (ARDC-Nectar Cloud, University of Queensland, QCIF, University of Melbourne and Pawsey) who underpin the Galaxy Australia service.

How were the tools and workflows selected?

The tool list here includes those that have been identified through BioCommons Community consultations and other engagements, as well as through matching to the tool sets installed across the various BioCommons partner compute infrastructures. The list is not intended to be exhaustive: i.e. it does not contain ALL bioinformatics tools.

How was the list generated?

RStudio (see session info at the bottom of this page) was used for processing.

The broad steps are described here:

  1. The manually curated tool list and separate tool lists provided by the BioCommons partner infrastructures were parsed (Galaxy Australia, NCI, Pawsey and QRISCloud - RCC).
  2. The complete set of tool lists were joined using the toolID as a key, and
  3. Links were embedded, where available, to the original documentation URLs (homepage or GitHub repository, for example) as well as the registry entries on bio.tools and BioContainers.
  4. Galaxy wrapping often entails that tool suites are separated into their individual tool components. As such, if a Galaxy tool was matched to the manually curated tool list, a tool shed search link was embedded in the Available in Galaxy toolshed column. This allows a user to search for the tool using a relevant search term (e.g. vcftools) to identify all the component tools available in the toolshed.
